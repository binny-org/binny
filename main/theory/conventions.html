Conventions#

This page summarises the notation and conventions used throughout the Binny documentation and API. The goal is to keep the mathematical notation used in the documentation consistent with the variable names used in the code.

Notation#

Throughout the documentation we use the following mathematical symbols.

Redshift#

  • Redshift is denoted by \(z\).

  • A redshift grid used for evaluating distributions is written as

    \[z = \{z_0, z_1, \ldots, z_N\}\]

  • All tomographic quantities in Binny are evaluated on a true redshift grid \(z\).

Redshift terminology#

Two types of redshift estimates commonly appear in tomographic analyses.

Spectroscopic redshift (spec-z)

  • Measured from spectral lines

  • Typically very precise

  • Often treated as the true redshift \(z\) in analyses

It is commonly abbreviated

  • specz in code

  • spec-z in text

Photometric redshift (photo-z)

  • Estimated from broadband photometry

  • Subject to measurement uncertainty and scatter

  • Often used for defining tomographic bins

Common abbreviations include

  • photoz in code

  • photo-z in text

  • pz in some cosmology literature

True vs observed redshift#

Tomographic binning often distinguishes between

  • true redshift \(z\)

  • observed photometric redshift \(z_{\mathrm{obs}}\)

Photometric binning therefore typically involves a photo-z error model

\[p(z_{\mathrm{obs}} \mid z)\]

which describes the probability of observing \(z_{\mathrm{obs}}\) given a galaxy with true redshift \(z\).

In Binny, tomographic bin distributions are always returned as functions of the true redshift grid \(z\), even when bins are defined using photometric redshift selections.

Redshift distributions#

  • The redshift distribution of a parent galaxy sample is

    \[n(z)\]

  • Tomographic bin distributions are written

    \[n_i(z)\]

    These represent the contribution of bin \(i\) to the parent distribution \(n(z)\), where \(i\) denotes the bin index.

  • When normalized,

    \[\int n(z)\,\mathrm{d}z = 1\]

    and

    \[\int n_i(z)\,\mathrm{d}z\]

    represents the fraction of galaxies contained in bin \(i\).

Bin edges#

Tomographic bins are defined by redshift boundaries

\[z_i^{\mathrm{min}}, \quad z_i^{\mathrm{max}}\]

such that the bin window is

\[W_i(z)\]

Selection windows#

The bin selection function is denoted

\[W_i(z)\]

For example, in the case of ideal spectroscopic bins,

\[\begin{split}W_i(z) = \begin{cases} 1 & z_i^{\mathrm{min}} \le z < z_i^{\mathrm{max}} \\ 0 & \text{otherwise} \end{cases}\end{split}\]

In photometric tomography this window becomes a probabilistic assignment kernel.

Bin indices#

Tomographic bins are indexed by

\[i = 0, 1, \ldots, N_{\mathrm{bins}} - 1\]

This matches the indexing used in Python arrays and dictionaries.

Throughout the documentation:

  • \(i\) denotes a row index

  • \(j\) denotes a column index

Correlation ordering#

When correlations between bins are computed, Binny adopts the convention

\[i \le j\]

meaning that only the upper triangle of the correlation matrix is typically used.

This is common in cosmology where correlations such as

\[C_\ell^{ij}\]

are symmetric,

\[C_\ell^{ij} = C_\ell^{ji}\]

and therefore only the upper triangle needs to be evaluated or stored.

When matrices are visualised, however, the full symmetric matrix may be shown for clarity.

Sample naming#

In cosmology analyses the terms lens and source have specific physical meaning. In Binny we distinguish between two levels of naming.

Generic bin sets#

For general tomography examples we refer simply to

  • bins

  • bin sets

  • tomographic bins

without assuming a specific cosmological role.

Cosmology-specific naming#

When used in a large-scale structure context the two most common galaxy samples are

Lens sample

  • foreground galaxies used for clustering or galaxy–galaxy lensing

Source sample

  • background galaxies used for weak lensing measurements

API argument names#

Function arguments in Binny use plain descriptive names rather than mathematical symbols.

For example:

Argument name

Mathematical meaning

z

\(z\)

nz

\(n(z)\)

bin_edges

\(z_i^{\mathrm{min}}, z_i^{\mathrm{max}}\)

bins

\(n_i(z)\)

n_bins

\(N_{\mathrm{bins}}\)

The documentation may use the mathematical notation for clarity, while the API consistently uses readable argument names.

Data structures#

Tomographic bins returned by Binny typically follow the structure

bins = {
    0: array([...]),
    1: array([...]),
    2: array([...]),
}

where

  • the dictionary key is the bin index

  • the value is the bin distribution evaluated on the redshift grid

Matrices#

Diagnostics that compare bins (overlap matrices, leakage matrices, correlation matrices) are written mathematically as

\[M_{ij}\]

and correspond in Python to nested dictionaries or arrays of the form

matrix[i][j]

Normalization conventions#

Unless otherwise specified:

  • parent redshift distributions are normalized

    \[\int n(z) \mathrm{d}z = 1\]

  • tomographic bins preserve the total number of galaxies

    \[\sum_i \int n_i(z) \mathrm{d}z = 1\]

Redshift grids#

All distributions returned by Binny are evaluated on the same input redshift grid provided by the user.

For example

z = np.linspace(0, 2, 500)

This ensures that

  • bins can be compared directly

  • diagnostic matrices can be computed consistently

  • plotting functions operate on aligned arrays.

Cosmology analysis jargon#

Large-scale structure and weak-lensing analyses often use shorthand terminology for common combinations of observables.

Weak lensing (WL)

  • distortion of background galaxy shapes by gravitational lensing

  • often used interchangeably with cosmic shear in tomographic analyses

Cosmic shear

  • weak-lensing correlations measured between source galaxy bins

  • typically written as shear–shear correlations

  • often referred to as shear

    \[C_\ell^{\gamma\gamma,\,ij}\]

or simply

\[C_\ell^{ij}\]

when the observable is clear from context.

Galaxy–galaxy lensing (GGL)

  • cross-correlation between lens galaxies and source galaxy shapes

  • measures how foreground galaxies trace the surrounding matter field

    \[C_\ell^{g\gamma,\,ij}\]

where

  • \(i\) labels a lens bin

  • \(j\) labels a source bin

Galaxy clustering

  • angular correlations of galaxies within the lens sample

    \[C_\ell^{gg,\,ij}\]

These correlations probe the spatial clustering of galaxies and their relation to the underlying matter distribution.

Combined two-point analyses#

Modern cosmology analyses often combine multiple two-point observables. Common shorthand names include:

3x2pt

Joint analysis of three types of two-point correlations:

  • galaxy clustering

  • galaxy–galaxy lensing

  • cosmic shear

Symbolically this corresponds to

\[\{C_\ell^{gg},\ C_\ell^{g\gamma},\ C_\ell^{\gamma\gamma}\}\]

This is the standard observable set used in many modern survey analyses.

2x2pt

Joint analysis using two of the three observables above. Common examples include:

  • cosmic shear + galaxy clustering

    \[\{C_\ell^{gg},\ C_\ell^{\gamma\gamma}\}\]

  • galaxy clustering + galaxy–galaxy lensing

    \[\{C_\ell^{gg},\ C_\ell^{g\gamma}\}\]

The exact combination is only a shorthand and should be defined clearly in each context. Most often, 2x2pt refers to the combination of galaxy clustering and galaxy–galaxy lensing, while cosmic shear is reserved for the 3x2pt case.

Bin roles in these analyses#

In this terminology:

  • lens bins refer to bins used for galaxy clustering and galaxy–galaxy lensing

  • source bins refer to bins used for weak lensing / cosmic shear

Thus:

  • clustering correlations use lens–lens bin pairs

  • galaxy–galaxy lensing uses lens–source bin pairs

  • cosmic shear uses source–source bin pairs
